# CI security guardrails

Этот документ фиксирует минимальные требования безопасности для CI-пайплайнов в репозитории.

## Базовые правила
- Не коммитим секреты в git: токены, ключи, реальные `.env`.
- Секреты передаются только через GitHub Actions Secrets / внешние secret storage (например, Lockbox).
- В шагах, где есть доступ к секретам, запрещено включать трассировку команд (`set -x`).
- Для shell-скриптов используем `set -euo pipefail`; при этом перед секретными операциями оставляем `set +x`.
- Логи не должны печатать значения переменных с секретами.

## Практика для GitHub Actions
- Использовать встроенное маскирование `secrets.*` и не делать `echo` секретов.
- Если секрет попадает в переменную, не печатать окружение целиком (`env`, `printenv`) в job с секретами.
- Для отладки использовать безопасные маркеры (`[redacted]`) вместо реальных значений.

## Проверочный чек-лист для PR
1. Нет новых файлов с реальными секретами (`.env`, ключи, токены).
2. В workflow нет `set -x` в шагах, где используются секреты.
3. В workflow нет команд, печатающих секреты в лог.
